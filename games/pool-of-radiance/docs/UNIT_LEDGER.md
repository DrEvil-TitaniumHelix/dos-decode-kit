## Role labelling — COMPLETE (was the "endurance slog" open item)

Every unit now carries an **evidence-backed role**, assigned by the static analyzer
`analysis/unit_roles.py`: it disassembles each unit's public entry points, resolves far-calls to
named resident segments, tallies state-global memory accesses, and scores the unit against role
categories. Output: `analysis/unit_roles.json`. The labels reproduce every previously
hand-verified unit (0x38=character state, 0x86=UI — both matched independently).

### ⭐ The graphics-engine split (was the leftover open item)
The generic GFX segment `0x709` serves **three distinct jobs**, told apart by call offset:
`0x1df` writes the VGA Sequencer **Map-Mask port 0x3c4** (planar pixel blit), `0xc3f` does
pixel-span scaling, while `0x626`/`0x768` touch `window_state 0x5e25` and compare against the
space char `0x20` (text/window paint). Applying that:
- **GfxBlit/Sprite** = `0x3b0` (the planar bitmap blitter — sprites + wall tiles) and `0x17`.
- **TextWindowPaint** = `0x6c, 0x77, 0x1e, 0x7d, 0x75, 0x23` (the UI compositors).
- **Compute/Logic** = `0x3c6` (70 `shl`, 86 `cwde`, 54 `les`, zero draw primitives = record /
  coordinate math).

**Finding: there is no real-time 3-D math engine.** No unit pairs heavy perspective arithmetic
with the pixel blitter. The first-person dungeon view is **pre-drawn wall-tile bitmaps blitted
per distance band by `0x3b0`** — consistent with the decoded 8X8D/WALLDEF wall tiles and with how
SSI's Gold Box engine actually worked. That closes the "which unit is the 3-D viewport" question:
none — it's bitmap compositing, not geometry.

### Notes on confidence
- **High** (signal unambiguous): the ECL/Events units (0x19, 0x1c), Combat/Encounter (0x68,
  0x3b4, 0x25), all ResourceLoad units, and the two hand-verified cores (0x38 CharacterState,
  0x86 UI/Window).
- **High** (now resolved by the 0x709-offset split): `GfxBlit/Sprite` (0x3b0, 0x17) confirmed by
  VGA Map-Mask port writes; `TextWindowPaint` (0x6c, 0x77, 0x1e, 0x7d, 0x75, 0x23) confirmed by the
  window-state + space-char paint primitives; `Compute/Logic` (0x3c6) by arithmetic density with
  zero draw calls.
- **InternalHelperLib**: high func-count / ≤3 entries and no feature signal = leaf utility code
  called by other units, not a feature owner.
- "Render/Display" now appears only as a *runner-up* hint in the table — every primary role is a
  specific bucket.
